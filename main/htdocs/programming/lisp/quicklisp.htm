Quicklisp

Lisp's least bad packaging system yet (TM). It is a dependency-aware repository of libraries. You can specify a library that you want installed, and Quicklisp will handle the dependencies. Think apt-get for Lisp.

Getting started

(load "quicklisp.lisp")
(quicklisp-quickstart:install)

(ql:quickload "system-name") ; load a system
(ql:register-local-projects) ; sorta
(ql:system-apropos "term") ; to find systems
(ql:update-client) ; new version takes effect upon restart 
(ql:update-dist "quicklisp") ; upgrade distribution

Interesting libraries

  • cl-markdown - a markdown processor
  • data-table - providing a data-table data-structure that has rows of data and column names and types. includes routines for reading/writing, converting, combining and sub-tabling. Easy interation with databases, and generating complex reports that contain multiple sections.
  • folio - features AS (type conversion), boxes, and collections.
  • optima - fast pattern matching library. (match "foo" ("foo" "bar")) => "bar". Also allows for place-patterns, guard-patterns, and other patterns
  • repl-utilities - ease life at the repl
  • trivial-project-pathname - resolving pathnames relative to some directory, usually the sources
  • trivial-shell - independent interface to the OS. It includes get-env-var, os-process-id, shell-command.

    Tips

    To load quicklisp every time you start lisp, type (ql:add-to-init-file).

    (ql:qsystem-list) works similarly to system-apropos, I think.

    (ql:system-apropos "") will list all the packages you can install. You can also view a it on this webpage.
